Custom Builds

Custom builds make it easy to create lightweight versions of Lodash containing only the features you need. To top it off, we handle all function dependency & alias mapping for you. Review the build differences & pick the one that’s right for you.

The Lodash command-line interface is available when lodash-cli is installed as a global package: {% raw %} 

$ npm i -g npm
$ npm i -g lodash-cli
$ lodash -h

{% endraw %}

Build Types

  • Core builds, that are 4 kB, are created using the core modifier. {% raw %} 
    lodash core
    {% endraw %}
  • Strict builds, with ES strict mode enabled, are created using the strict modifier. {% raw %} 
    lodash strict
    {% endraw %}
  • Modularized builds, with Lodash split into modules, are created using the modularize modifier. {% raw %} 
    lodash modularize
    {% endraw %}

Build Commands:

  • Use the category command to pass comma separated categories of functions to include in the build. Valid categories are “array”, “collection”, “date”, “function”, “lang”, “object”, “number”, “seq”, “string”, & “util”. {% raw %} 
    lodash category=collection,function
    {% endraw %}
  • Use the exports command to pass comma separated names of ways to export the lodash function. Valid exports are “amd”, “commonjs”, “es”, “global”, “node”, “npm”, “none”, & “umd”. {% raw %} 
    lodash exports=amd,node
    {% endraw %}
  • Use the iife command to specify code to replace the IIFE that wraps Lodash. {% raw %} 
    lodash iife="\!function(window,undefined){%output%}(this)"
    {% endraw %}
  • Use the include command to pass comma separated names of functions to include in the build. {% raw %} 
    lodash include=each,filter,map
    {% endraw %}
  • Use the minus command to pass comma separated function/category names to remove from the build. {% raw %} 
    lodash minus=result,shuffle
    {% endraw %}
  • Use the plus command to pass comma separated function/category names to add to the build. {% raw %} 
    lodash category=array plus=random,template
    {% endraw %}
  • Use the template command to pass the file path pattern used to match template files to precompile. Note: Precompiled templates are assigned to the _.templates object. {% raw %} 
    lodash template="./*.jst"
    {% endraw %}
  • Use the settings command to pass template settings used when precompiling templates. {% raw %} 
    lodash settings="{interpolate:/\{\{([sS]+?)\}\}/g}"
    {% endraw %}
  • Use the moduleId command to specify the AMD module ID for Lodash or the module ID used to include Lodash in compiled templates. Use “none” as the module ID to create compiled templates without a dependency on Lodash. {% raw %} 
    lodash moduleId=underscore
    {% endraw %}

Notes:

  • The exports values “es” & “npm” may only be used in conjunction with the modularize command
  • The modularize command uses the first exports values as its module format, ignoring subsequent values
  • Unless specified by -o or --output all files created are saved to the current working directory

The following options are also supported:

  • -c, --stdout ................ Write output to standard output
  • -d, --development ..... Write only the non-minified development output
  • -h, --help .................... Display help information
  • -m, --source-map ....... Generate a source map using an optional source map URL
  • -o, --output ................ Write output to a given path/filename
  • -p, --production ....... Write only the minified production output
  • -s, --silent ............... Skip status updates normally logged to the console
  • -V, --version ............. Output current version of Lodash
